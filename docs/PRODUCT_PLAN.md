### 1.2 产品定位

```
         恐慌制造者                        权威数据源
    (国外竞品)                            (WHO/CDC)
         |                                    |
         |    ★ 汉坦观察的定位                 |
         |    ←────────────────────→           |
         |    科学克制 + 用户友好 + 移动优先     |
         |                                    |
    完全娱乐化                           完全学术化
    (自媒体)                              (学术论文)
```

**流行病学背景（2026-05）：**
- 中国每年报告 HFRS（肾综合征出血热，病原体主要为汉滩型/汉城型）约 1-2 万例，属**地方性流行（endemic）**，发病数长期稳定，非新兴公共卫生事件。
- 近期引发全球媒体关注的 Andes 病毒聚集性疫情（MV Hondius 邮轮，2026 年 5 月，6-7 例确诊，3 例死亡）发生在南美洲海域，与中国直线距离超过 15,000 公里。**Andes 病毒具备人际传播能力，这是其区别于中国地方性汉滩/汉城型的关键差异。**
- WHO / CDC / ECDC 对 Andes 邮轮疫情的评估均为"对公众风险低"。目前全球未发现邮轮外社区传播。
- **产品核心价值：** 在"远处有事、近处平静"的信息差中，提供准确、透明、不恐慌的态势感知。

**一句话定位：** 面向中文用户的移动优先型汉坦病毒预警与信息平台。以透明方法论提供"病毒离我们有多远"的距离监测、中国专属风险评估、以及基于官方数据的预警推送。

**差异化锚点（vs 国外恐慌型竞品）：**
1. **病毒血清型分离展示** — 汉滩型(Hantaan)、汉城型(Seoul)、普马拉型(Puumala)、辛诺柏型(Sin Nombre)、安第斯型(Andes) 各自独立数据面板
2. **数据溯源 100%** — 每条数据标注原始出处 URL、发布时间、数据口径
3. **工具专属 HPI 汉坦逼近指数** — 在引用 WHO/CDC 官方评估的同时，提供工具自有的、方法论完全透明的中国专属风险评分（Hanta Proximity Index），综合距离因子、官方评估、季节性、旅行联通度等维度，每个因子可独立检验
4. **"暂无数据"比假数据好** — 宁可显示空白也不做预测外推
5. **教育前置** — 用户看到数据前先理解"这是什么、怎么传播、如何防护"

#### 核心功能详述

**F0 · 距离仪表盘** ★ 核心差异化功能
- 实时计算并展示"最近的汉坦病毒活跃聚集地距离中国边境多远"
- 视觉呈现：雷达式同心圆仪表盘，中国为中心
  - 内圈（0-500km）：红色警戒区
  - 中圈（500-3000km）：橙色关注区
  - 外圈（3000km+）：绿色安全区
- 当最近的活跃聚集地距离变化时触发预警
- 距离数据的计算依据：WHO 疫情通报中的地理位置 + 血清型活跃状态
- **当前状态示例（2026-05）：** Andes 病毒聚集地（MV Hondius，南美洲）→ 距离中国边境 >15,000km → 安全区

**F1 · 病例地图**
- 交互式地图，省级 / 地市级粒度（国内）+ 国家级粒度（全球）
- 不同血清型用不同颜色标记（汉滩型=红，汉城型=橙，普马拉型=蓝，Andes型=紫，其他=灰）
- 点击区域弹出卡片：病例数、血清型、数据来源、最后更新时间
- **关键设计：** 地图默认以中国为中心展示，不自动聚合全球数据制造"全球大流行"假象；全球视图需手动切换

**F2 · 趋势图表**
- 时间序列折线图：月度/年度病例数变化
- 同期对比：今年 vs 去年 vs 5年均值
- 血清型分离曲线（各型别独立线条）
- 标注关键事件节点（如"张家口鼠疫监测加强"等学术/政策事件）

**F3 · 血清型分类面板**
- 每种汉坦病毒血清型的独立信息卡：
  - 主要宿主（黑线姬鼠 / 褐家鼠 / 岸䶄 / 鹿鼠 等）
  - 传播途径（气溶胶 / 接触 / 人传人？）
  - 病死率范围
  - 主要分布区域
  - 是否具备人传人能力（★ 这是国外竞品故意混淆的点）
- 用户可在不同血清型间切换对比

**F4 · 数据溯源面板**
- 每条数据的"数据身份证"：
  - 来源机构 + 原始链接
  - 发布时间
  - 数据口径说明（"确诊病例""临床诊断病例""监测点报告"）
- 数据置信度标识（官方通报 / 学术文献 / 媒体报道 / 未经证实）

**F5 · 双重风险评估**
- **左侧：官方评估原文** — 直接嵌入 WHO / CDC / 中国疾控中心的最新风险评估原文摘要，标注引用日期，不做二次解读
- **右侧：HPI 汉坦逼近指数** — 工具自有的中国专属风险评分（详见 §1.6），方法论完全透明，每个因子可独立检验
- 二者并列展示，让用户同时看到"国际权威怎么说"和"我们的多因子分析怎么说"
- 当 HPI 与官方评估出现方向性差异时，自动添加说明文字解释差异来源

**F6 · 教育内容**
- 病毒百科：汉坦病毒的生物学、流行病学基础
- 防护指南：防鼠灭鼠、野外作业防护、早期症状识别
- 辟谣专区：针对社交媒体常见谣言做事实核查
- Q&A：用户提问 → 人工审核后发布回答

**F7 · 预警推送** ★ MVP 核心功能
- 订阅触发 - 用户可选：全国 / 特定省份 / 特定血清型
- 自动触发条件：
  1. HPI 指数跨越阈值（如从"低关注"升至"中等关注"）
  2. 距离仪表盘：活跃聚集地跨越距离圈层（如从外圈进入中圈）
  3. 官方发布新病例通报
  4. 新血清型首次在邻近国家/国内出现
- 推送文案克制，不渲染恐慌："汉坦观察提醒：Andes病毒聚集地距中国边境距离缩短至 X,XXX km（进入关注区）。HPI指数：35/100（中等关注）。点击查看详情。"
- 支持微信服务号模板消息 + 浏览器 Web Push

**F8 · 分享裂变**
- 一键生成9:16竖版海报（适配抖音/小红书）
- 海报模板：知识卡片型（如"5个关于汉坦病毒的真相"）、数据卡片型（如"你所在城市的出血热风险"）
- 海报底部嵌入小程序码/二维码回流

### 1.5 设计原则

1. **移动端优先 (Mobile-First)**
   - 所有页面以 375px 宽度为基准设计
   - 关键操作单手可达（拇指热区）
   - 图表在移动端的降级方案（横屏提示 / 简化视图）
   - 加载性能目标：First Contentful Paint < 1.5s（4G 网络）

2. **科学克制 (Scientific Restraint)**
   - 调色板：白底 + 深蓝主色 + 低饱和辅助色（避免红色滥用）
   - 不使用脉冲动画、"LIVE"标识、闪烁警告
   - 不确定性用灰色/虚线表达，确定数据才用实色
   - 数字精确到个位，不取近似值制造"整数恐慌"

3. **可溯源性 (Traceability)**
   - 每个数据点可点击查看来源
   - 来源链接直接跳转至原始报告（非二次转载）
   - 数据更新时保留历史快照（用户可追溯变更）

4. **无障碍 (A11y)**
   - WCAG 2.1 AA 级别
   - 支持屏幕阅读器
   - 色盲友好配色方案

### 1.6 HPI 汉坦逼近指数 (Hanta Proximity Index)

> HPI 是汉坦观察自有的中国专属风险评分。与国外竞品的"黑箱评分"不同，HPI 的每一个因子、权重、数据来源均公开可查。用户可以在 `/about` 页面查看完整计算公式和实时计算日志。

#### 评分框架

```
HPI = W₁×D + W₂×O + W₃×S + W₄×T + W₅×H

  因子          权重    说明
  ───────────  ──────  ──────────────────────────────────────
  D 距离因子    30%    最近活跃聚集地距中国边境的距离（km）
                       分级：>10,000km=0分 / 3,000-10,000=20分
                       500-3,000=50分 / <500km=100分
  O 官方评估    25%    WHO/CDC/中国CDC 最新风险评估的量化映射
                       "低风险"=0 / "中等"=40 / "高"=70 / "极高"=100
  S 血清型风险  20%    当前活跃血清型的人际传播能力 + 病死率
                       Andes型(有人传人+高病死率)=100
                       汉滩型(无人传人，中病死率)=30
                       汉城型(无人传人，低病死率)=20
                       普马拉型(无人传人，极低病死率)=5
  T 旅行联通度  15%    活跃聚集地与中国之间的航班/陆路联通密度
                       直飞航线=高分 / 需转机=中分 / 无直连=低分
  H 历史基线    10%    中国本地 HFRS 发病数 vs 历年同期均值
                       超均值50%以上=90 / 正常范围=20 / 低于均值=0
```

#### 评分等级

| HPI 范围 | 等级 | 颜色 | 含义 | 用户行动建议 |
|----------|------|------|------|-------------|
| 0-20 | 低关注 | 绿 | 无明显威胁 | 保持日常防护习惯 |
| 21-40 | 一般关注 | 蓝 | 有远距离聚集性疫情，但距中国遥远 | 关注更新，无需额外措施 |
| 41-60 | 中等关注 | 黄 | 疫情向中国邻近区域移动，或国内发病数异常 | 加强个人防护，关注官方通报 |
| 61-80 | 高度关注 | 橙 | 邻近国家出现人传人疫情，或国内出现异常聚集 | 避免高风险区域旅行，储备防护物资 |
| 81-100 | 严重关注 | 红 | 国内出现人际传播或大规模暴发 | 严格遵循疾控中心指导 |

#### 与国外竞品评分的核心差异

| 维度 | HPI 汉坦逼近指数 | 国外竞品"RISK 8.4" |
|------|-----------------|-------------------|
| 方法论 | 完全公开，每个因子可独立验算 | 不透明，无公开公式 |
| 视角 | 中国专属（距离因子以中国边境为基准） | 全球无差别评分 |
| 血清型区分 | 不同血清型赋予不同风险权重 | 所有汉坦病毒混为一谈 |
| 距离因子 | 明确量化（km → 分数映射） | 无距离考虑 |
| 官方评估 | 独立引用，权重仅 25% | 不引用官方评估 |
| 趋势 | 展示 HPI 历史变化趋势箭头 | 无历史趋势 |
| 数据溯源 | 每个因子的输入数据可点击查看来源 | 不可溯源 |

#### 当前 HPI 示例（2026-05-12）

```
距离因子 (D): Andes/MV Hondius >15,000km → 0 分 × 30% = 0
官方评估 (O): WHO low risk → 0 分 × 25% = 0
血清型风险(S): Andes型 = 100 × 20% = 20
旅行联通度(T): 南美乌斯怀亚-中国, 无直飞, 需2次转机 → 10 分 × 15% = 1.5
历史基线 (H): 中国HFRS处于正常范围 → 20 分 × 10% = 2
─────────────────────────────────────────────
HPI = 23.5 → 等级：一般关注（蓝）

解释：尽管 Andes 病毒本身危险性高（人传人+高病死率），
但因距中国极远（>15,000km）且 WHO 评估公众风险低，
当前对中国的实际威胁有限。HPI 的 20 分主要来自血清型
本身的风险属性（非地理邻近性）。
```

> **设计原则：** HPI 不替代官方评估，而是作为中国视角的补充。当 HPI 与 WHO/CDC 评估出现方向性差异时，界面必须展示差异来源的透明解释。算法代码开源，任何研究者可复现验算。

| 层级 | 技术 | 选型理由 |

| **数据库** | PostgreSQL 16 + PostGIS | 地理空间数据原生支持；JSONB 灵活存储半结构化数据 |
| **缓存** | Redis | 热点数据缓存；API 限流 |
| **搜索** | MeiliSearch (开源) | 中文分词支持好；部署轻量 |
| **CI/CD** | GitHub Actions | 免费额度足够；社区生态丰富 |
| **部署** | 阿里云 / Vercel (前端) + 阿里云 ECS (后端) | 国内用户访问速度；Vercel 边缘部署前端静态资源 |
| **监控** | Sentry (错误跟踪) + 自建 Uptime | 实时错误告警；服务可用性监控 |
| **CDN** | 阿里云 CDN + 又拍云 | 国内多节点加速 |

### 2.3 技术决策关键考量

- **为什么不用 Python 做后端？** — 数据采集模块用 Python（生态最优），但 API 服务用 Go/Rust（性能 + 类型安全），两者分离。
- **为什么不用 Vue？** — Next.js 的 SSR/ISR 对 SEO 更友好，且 App Router 模式更现代。
- **为什么保留小程序？** — 微信小程序是用户触达最短路径；Taro 可以一套代码编译到微信 + 抖音小程序。
- **为什么用 MapLibre 而非高德地图？** — 地图数据的完全控制权；可以自定义瓦片样式避免"红色恐慌地图"；同时保留高德地图 JS API 作为国内降级方案。

### 4.3 数据质量控制

- **血清型分类校验**：自动匹配宿主物种 → 血清型映射表，人工二次确认
- **地理编码校验**：抓取的地名 → 标准行政区划代码（GB/T 2260）
- **去重逻辑**：同一病例被多源报道 → 按"病例 ID = 报告地区 + 确诊日期 + 血清型 + 性别年龄"去重
- **时间异常检测**：报告日期 > 当前日期 → 自动标记为异常待审
- **数值突变检测**：月度病例数环比增幅 > 300% → 自动标记待审
- **历史快照**：每次数据更新保留变更记录，支持回溯

### 5.2 GEO (生成式引擎优化 — 针对 AI 搜索/大模型)

> GEO 的核心目标是：当用户在 ChatGPT、Kimi、豆包、文心一言等 AI 工具中提问汉坦病毒相关内容时，本产品的内容能被引用。

| 策略 | 说明 |
|------|------|
| **结构化知识输出** | 关键数据以 JSON-LD + Markdown 表格双格式提供，方便 AI 解析 |
| **LLMs.txt 文件** | 根目录提供 `/llms.txt`，包含站点核心知识的结构化摘要，专供大模型读取 |
| **引用友好文案** | 关键事实用独立段落 + 明确归属（"据中国疾控中心2026年4月月报…"），便于 AI 引用时保留出处 |

| **FAQ 结构化** | Q&A 使用 `Question` / `Answer` 语义标记，提升在 AI 摘要中的曝光率 |
| **Z-Index 内容策略** | 首页直接展示核心数据+结论，而非引导点击 — AI 爬虫通常只抓首页前 5000 字符 |

## 8. 风险与合规

### 8.1 医疗信息合规

**中国法律法规依据：**
- 《互联网信息服务管理办法》
- 《医疗卫生信息服务管理办法》
- 《网络安全法》《数据安全法》《个人信息保护法》

**红线清单：**
- ❌ 不提供个体医疗建议（"你可能是感染了，快去医院"）
- ❌ 不推荐具体药物或治疗方案
- ❌ 不宣称可替代专业医疗机构
- ❌ 不发布未经官方确认的疫情预测
- ❌ 不制造社会恐慌
- ✅ 明确标注"本工具不提供医疗建议，如有症状请前往正规医疗机构就诊"
- ✅ 所有健康信息注明出处和更新时间
- ✅ 设立免责声明独立页面

### 8.2 数据安全

- 用户数据最小化收集（仅订阅推送需要：地区 + 推送 token）
- 前后端分离，API 鉴权（JWT）
- 数据库定期备份 + 异地灾备
- 爬虫遵守 robots.txt + 合理频率（不超过 1 req/s per domain）

### 8.3 免责声明（首页底部 + 独立页面）

> **免责声明：** 汉坦观察 (HantaWatch) 是一个信息聚合平台，旨在提供汉坦病毒相关的公共卫生信息。本工具不提供医疗诊断、治疗建议或个性化的健康指导。所有数据均来自公开的官方来源，但我们不保证数据的完整性、准确性和及时性。世界卫生组织 (WHO)、中国疾病预防控制中心及其他数据来源机构不对本工具的内容负责。如果您出现发热、头痛、腰痛、眼眶痛、面部潮红等疑似症状，请立即前往正规医疗机构就诊。

### 8.4 危机预案

| 场景 | 应对方案 |
|------|----------|
| 国内出现汉坦病毒社区传播 | 立即启动应急更新流程，首页添加实时数据面板，暂停所有娱乐化内容 |
| 卫健委/网信办认定内容违规 | 即时下架争议内容，主动联系监管机构沟通整改方案 |
| 数据被媒体误用/曲解 | 发布澄清声明，联系相关媒体更正，必要时法律手段 |
| 网站被攻击/数据篡改 | 自动回滚到上一个安全版本，安全审计后重新上线 |
| 社交媒体出现大规模恐慌 | 发布冷静声明，引用 WHO/CDC 官方风险评估，关闭评论区 |
