# 数据链修复日志

> 日期：2026-05-17 | 关联审计：`docs/DATA_CHAIN_AUDIT.md`

## 原则

- 展示"最近威胁距离"（输入病例比源头更近时显示输入距离）的组件：主距离卡片、底部指标数字、分享文案、海报
- 展示"疫情源头距离"（永远显示 Andes 爆发源地距离）的组件：NearestAndesCard 及其内部 DistanceBar、源头标注文字
- **源头距离 16,500km 保持不变；仅将"该显示威胁距离却显示了源头距离"的地方修正**

## 修复记录

### Fix 1: MiniApp "3原子数字"行 — 距中国大陆指标

**文件：** `apps/miniapp/src/pages/home/index.tsx` L295

### Fix 2: MiniApp 微信分享文案

**文件：** `apps/miniapp/src/pages/home/index.tsx` L67-74  
**改动：** `useShareAppMessage` 回调中增加 `hasImportDistance` 判断，有输入病例时显示 `"汉坦距中国大陆 X,XXX km（西班牙输入）"` 而非 Andes 源头距离。  
**效果：** 用户分享卡片时显示的是实际威胁距离。

### Fix 3: Web "3原子数字"行 — 距中国大陆指标

**文件：** `apps/web/src/app/page.tsx` L343-344  
**改动：** `cluster.distanceFromChinaKm` → `displayedDistanceKm`；标签增加 `hasImportDistance` 条件。  
**效果：** 与 Fix 1 相同逻辑，Web 端同步修复。

### Fix 4: Web 分享海报 API

**文件：** `apps/web/src/app/api/poster/route.tsx`  
**改动：**
- L6：新增 import `hondiusImports`、`findNearestImport`、`ImportRecord`
- L44-49：新增 `nearestImport` / `hasImport` / `distanceKm` / `distanceLabel` 计算
- L155：标签从静态 `"最近聚集地距中国"` 改为动态 `{distanceLabel}`  
**效果：** 生成的海报距离数字与首页一致，有输入时显示"距最近输入 西班牙 9,200 km"。

### 未修改（保留源头距离）

以下位置有意保留 Andes 源头距离（16,500km），不做修改：

| 位置 | 原因 |
|------|------|
| `NearestAndesCard`（两端）| 卡片标题为"最近 Andes 型疫情"，展示的是疫情爆发源地距离 |
| `DistanceBar` 组件 | 接收调用方传入的距离，在 NearestAndesCard 中传入源头距离是正确的 |
| Hero 卡片 "疫情源头: 乌斯怀亚（16,500 km）" 标注 | 源头注释，展示源头距离是正确的 |
| `lib/nearest-cluster.ts` `findNearestBySerotype` | 纯数据函数，返回源头距离，由调用方决定如何展示 |

  - `country-status.json` 长期流行基线
