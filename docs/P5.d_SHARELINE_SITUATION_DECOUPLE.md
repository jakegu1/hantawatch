1. **One source of truth per fact.** A number printed in `shareLine` must not also appear in `situation`. If both sentences need to reference the same outbreak total, only `shareLine` carries the digit.

## 3. Behavior contract: shareLine vs situation

### 3.1 `shareLine` — "the screenshot sentence"

**Role:** the single line a user can screenshot and forward to a WeChat group. Must be self-contained.

**Required content (in this order):**
1. The WHO cumulative total + last-update date + lag indicator (handled by `_enforce_who_lag_disclosure`).
2. **All** confirmed-new additions surfaced via realtime LLM extractor since the WHO update (per-country, comma-separated, with the "待 WHO 复核" tag if `evidence[].tier == "news"`).
3. Mainland China status sentence (no local cases, HFRS baseline normal).

**Required style:**
- ≤ 80 Chinese characters.
- Direct factual phrasing. No editorializing, no "请大家关注".
- Numbers stay as Arabic digits.
- Compliant geographic naming per `@d:\Work\Hanta\services\collector\hantawatch_collector\_compliance.py`.

**Good example (target):**
```
WHO 5/13 公布累计 11 例（13 天前）；其后西班牙、法国各新增 1 例确诊输入，待 WHO 复核；中国大陆无相关病例，国内 HFRS 基线正常。
```

### 3.2 `situation` — "the context sentence"

**Role:** explain to the reader *why the numbers in `shareLine` look the way they do* and what is in flight.

**Required content (pick 2–3 of these, do not enumerate all):**
- How stale the WHO data is and what fills the gap (e.g. "WHO 数据每 1–2 周更新，近期由各国卫生部 / CDC 公告补足。").
- Which countries are in monitoring status vs which have confirmed imports (qualitative, no per-country counts).
- What is pending official confirmation (e.g. "西班牙、法国新增暂待 WHO 复核。").
- Whether the mainland baseline is changing (qualitative).

**Forbidden content:**
- Any digit that already appears in `shareLine` (cumulative total, per-country additions, lag day count, etc.). The post-processor enforces this via Jaccard similarity; the prompt must teach the LLM to avoid it.
- The exact phrase "WHO M/D 公布累计 N 例（K 天前）" — that belongs to `shareLine`.
- Repetition of the mainland-China clause already in `shareLine`. If `shareLine` says "中国大陆无相关病例，国内 HFRS 基线正常", `situation` may say "国内基线未变" but must not restate "中国大陆无相关病例".

**Required style:**
- ≤ 75 Chinese characters.
- Explanatory phrasing. May start with conjunctions like "WHO 数据滞后期间…", "其中…", "目前…".

**Good example (target, paired with the shareLine above):**
```
WHO 数据每 1–2 周更新，期间由各国卫生部公告与 ArcGIS 监测补足；西班牙新增暂待 WHO 复核，多国（NL/DE/GB 等）维持监测，国内基线未变。
```

### 3.3 The Jaccard guardrail

When the LLM ignores the role split (which it will, sometimes), the post-processor replaces `situation` with a deterministic fallback. This is a quality floor, not a quality ceiling — the prompt change in §4.1/§4.2 is the primary fix.

### 4.4 New: `_dedupe_share_situation`

**Signature:**

```python
def _dedupe_share_situation(
    brief: dict[str, Any],
    outbreak_status: list[dict[str, Any]] | None,
) -> tuple[dict[str, Any], list[str]]:
```

**Module-level constants (at top of `ai_brief.py`, near `SYSTEM_PROMPT`):**

```python
SHARE_SITUATION_JACCARD_THRESHOLD: float = 0.55
SHARE_SITUATION_NGRAM_SIZE: int = 2
```

**Helper (also pure, also module-level):**

```python
def _jaccard_char_bigrams(a: str, b: str, n: int = SHARE_SITUATION_NGRAM_SIZE) -> float:
    """Whitespace- and ASCII-punctuation-stripped char n-gram Jaccard.
    Returns 0.0 if either input is empty after normalization."""
```

Normalization rules inside `_jaccard_char_bigrams`:
- Strip whitespace and the following punctuation characters: `，。；：、（）()「」“”"',.;:!?` (both ASCII and full-width).
- Do **not** lowercase (Chinese is case-irrelevant; preserving ASCII case is safer).
- If either string is shorter than `n` characters after normalization, return `0.0`.

**Behavior of `_dedupe_share_situation`:**

1. Read `share = brief.get("shareLine") or ""` and `sit = brief.get("situation") or ""`.
2. If either is empty, return `(brief, [])` unchanged.
3. Compute `jaccard = _jaccard_char_bigrams(share, sit)`.
4. If `jaccard < SHARE_SITUATION_JACCARD_THRESHOLD`, return `(brief, [])` unchanged.
5. Otherwise, build the fallback (see below), set `brief["situation"] = fallback`, append a warning string of the form `f"share_situation_overlap: replaced (jaccard={jaccard:.2f})"`, and return `(brief, [warning])`.

**Fallback builder:**

```python
def _build_situation_fallback(
    outbreak_status: list[dict[str, Any]] | None,
) -> str:
```

- Inputs: the outbreak status ledger.
- Output: a single sentence ≤ 75 chars, derived deterministically from the ledger:

```
WHO 数据每 1–2 周更新，期间由各国卫生部公告与监测面板补足；{news_countries} 新增暂待 WHO 复核，多国维持监测，国内基线未变。
```

Where `{news_countries}` is the comma-joined Chinese names (`nameZh`) of countries in `outbreak_status[0].perCountry` whose `evidence` array contains at least one entry with `tier == "news"`. If none, use the template with the clause "近期暂无新增待复核" replacing the `{news_countries} 新增暂待 WHO 复核` segment.

If `outbreak_status` is empty or malformed, return the literal generic fallback:

```
WHO 数据每 1–2 周更新；当前依据各国卫生部公告与监测面板补足，国内基线未变。
```

**Integration in `enhance_daily_brief`:**

Insert the call **after** `_enforce_who_lag_disclosure` and **before** `_validate_brief_against_ledger`, around `@d:\Work\Hanta\services\collector\hantawatch_collector\ai_brief.py:468`:

```python

if dedup_warnings:
    existing = enhanced.get("_guardrail_warnings")
    merged = list(existing) if isinstance(existing, list) else []
    merged.extend(dedup_warnings)
    enhanced["_guardrail_warnings"] = merged
    logger.info("brief share/situation dedup: %s", "; ".join(dedup_warnings))
```

Do not reorder existing post-processing steps.
